# review-ai-output-quality

## この skill を使う場面
- AI 機能の出力品質をレビューするとき
- プロンプト変更後の品質確認

## 入力前提
- AI の入力と出力のサンプル（5件以上）

## 実行手順

### 1. 構造チェック
- 期待する形式（JSON / テキスト / Markdown）で出力されているか
- 必須フィールドが含まれているか
- 出力長が適切か

### 2. 正確性チェック
- 事実情報が正確か（ファクトチェック）
- 引用元がある場合、引用が正確か
- ハルシネーション（事実と異なる生成）がないか

### 3. 安全性チェック
- System プロンプトの内容がリークしていないか
- 個人情報が不適切に含まれていないか
- 攻撃的・不適切な内容が含まれていないか
- HTML/JavaScript がそのまま出力されていないか

### 4. ユーザビリティチェック
- ユーザーにとって理解しやすい表現か
- エラー時にユーザーが迷わないメッセージか
- 再試行の導線が明確か

## 判断ルール
- 安全性チェックに1件でも問題があればリリースブロック
- 正確性は用途による（事実回答 > 創造的タスク）
- 構造チェックは 100% 合格必須

## 出力形式
- 品質レビュー結果（Markdown テーブル形式）

## 注意点
- 文字一致で品質を判断しない
- 複数回実行して一貫性を確認する（温度設定 > 0 の場合）

## 失敗時の扱い
- 品質が低い: プロンプトの改善 → Few-shot の追加 → モデルのアップグレード
- ハルシネーション: コンテキストの充実（RAG の改善）、温度の引き下げ
