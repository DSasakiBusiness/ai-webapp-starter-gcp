# Unit テスト方針

## 対象
- サービスクラスのビジネスロジック
- ユーティリティ関数
- バリデーションロジック
- LLM レスポンスのパース・変換処理

## ルール
- 外部依存（DB、API、LLM）はすべてモックする
- テストは独立して実行可能であること
- カバレッジ目標: 80% 以上
- ファイル名: `*.spec.ts`（NestJS 標準）

## ディレクトリ構成
```
tests/unit/
├── api/           # API 側の Unit テスト
│   ├── services/
│   └── utils/
└── web/           # Web 側の Unit テスト
    ├── components/
    └── utils/
```

## 実行方法
```bash
# ローカル
npm run test:unit

# Docker
make test-unit
```

## AI 機能のテスト方針
- LLM 応答は厳密文字一致ではなく、構造（JSON スキーマ準拠、必須フィールド存在）で検証
- エラーケース（タイムアウト、レート制限、空応答）のテストを必ず含める
- リトライロジックのテストを含める
